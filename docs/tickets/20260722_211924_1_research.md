# Technical Research: Issue 1.1: Next.js App Router & Base Folder Scaffolding (Issue #1)

## Executive Summary
This is a greenfield scaffolding task to initialize a Next.js 14+/15 App Router project with TypeScript, ESLint, Vanilla CSS/CSS Modules (no Tailwind), and `@/*` import alias, then create the standard folder structure (`/app`, `/components`, `/services`, `/lib`, `/types`, `/tests`) and replace default boilerplate with a clean MVP landing page. The repository is currently empty (no `package.json`, `tsconfig.json`, or source code) aside from a Next.js-aware `.gitignore`. The issue is well-specified, has no external dependencies, and is ready to implement.

## Technical Challenges
- **Non-interactive CLI invocation**: `create-next-app@latest` when invoked with flags must fully suppress all interactive prompts. A March 2026 fix to create-next-app ensures passing **any** flag skips prompts and applies defaults for unspecified options. The exact flag set must be correct to avoid accidental Tailwind inclusion (Tailwind is the default).
- **Folder structure alignment**: The issue specifies root-level directories (`/app`, `/components`, etc.) without a `src/` directory. The `create-next-app` `--no-src-dir` default already places `app/` at root level, which matches. The custom folders (`/services`, `/lib`, `/types`, `/tests`) are not created by the CLI and must be created manually.
