# Implementation Plan: 6.1b: Camera Viewfinder, Capture Overlay & Coordinator Component (Issue #39)

## Executive Summary

This implementation plan details the creation of three interactive camera user interface components (`CameraViewfinder`, `CaptureOverlay`, and `CameraCaptureModal`) along with updating barrel exports and writing unit/component tests. The implementation integrates existing helpers (`lib/camera.ts`, `lib/location.ts`) and UI elements (`GlassCard`, `GlassPanel`, `Skeleton`) to provide a responsive wildlife photo capture experience with concurrent camera and geolocation initialization, stream lifecycle management, and multipart `FormData` API submission to `/api/capture`.

1. **`CameraViewfinder` Component**:
   - File: [`components/camera/CameraViewfinder.tsx`](file:///Users/jamescraig/gh/gregduck/components/camera/CameraViewfinder.tsx)
   - Function Signature:
     ```typescript
     export interface CameraViewfinderProps {
       stream: MediaStream | null;
       videoRef: React.RefObject<HTMLVideoElement | null>;
       isLoading?: boolean;
       error?: string | null;
       className?: string;
     }

     export const CameraViewfinder: React.FC<CameraViewfinderProps> = {
       stream,
       videoRef,
       isLoading = false,
       error = null,
       className = '',
     };
     ```
   - Logic:
     - Binds `videoRef.current.srcObject = stream` via `useEffect` whenever `stream` changes.
     - Displays `Skeleton` loading component when `isLoading` is true or `stream` is null without error.
     - Displays error alert UI when `error` is present.
     - Renders `<video autoPlay playsInline muted>` wrapped in `<GlassCard hoverEffect={false}>`.

3. **`CameraCaptureModal` Coordinator Component**:
   - File: [`components/camera/CameraCaptureModal.tsx`](file:///Users/jamescraig/gh/gregduck/components/camera/CameraCaptureModal.tsx)
   - Function Signature:
     ```typescript
     export interface CameraCaptureModalProps {
       isOpen: boolean;
       onClose: () => void;
       onSuccess?: (result: any) => void;
       onError?: (error: Error) => void;
     }

     export const CameraCaptureModal: React.FC<CameraCaptureModalProps> = {
       isOpen,
       onClose,
       onSuccess,
       onError,
     };
     ```
   - Logic:
     - Modal dialog wrapper (`role="dialog"`, `aria-modal="true"`) using `GlassPanel`.
     - When `isOpen` becomes true, concurrently initiates `requestCameraStream()` and `getCurrentLocation()`.
     - Maintains `streamRef` synchronized with `stream` state to guarantee `stopCameraStream(streamRef.current)` is invoked in `useEffect` cleanup upon close or unmount.
     - On capture trigger: calls `captureFrameAsBlob(videoRef.current)`, constructs `FormData` with `file`, `latitude`, `longitude`, POSTs to `/api/capture`, invokes `onSuccess(data)` and closes modal.
