# Technical Research: Issue 4.2: Proximity Matching Engine (Issue #13)

## Executive Summary

This research assesses the technical implementation requirements for building the proximity matching engine layer for Issue #13. The codebase already provides H3 resolution 9 spatial indexing helpers (`latLngToH3Index`, `getLocalZone`) and database schema definitions for animals. The issue is fully specified and ready to implement without external blockers or missing dependencies.

## Technical Challenges

- **Spatial Saturation Boundary Enforcement**: The proximity engine must determine species capacity limits within a 7-cell H3 resolution 9 neighborhood (1 center cell + 6 surrounding neighbors). Efficiently filtering existing animals across this 7-cell ring requires utilizing `getLocalZone` array lookups.
- **Distance Resolution Strategy**: When species count meets or exceeds saturation limit (defaulting to 3 per species), the engine must compute great-circle/Haversine physical distances between the target location and existing animal discoveries in the neighborhood to determine the closest match accurately.
- **Normalized Species Matching**: Species strings must be normalized (case-insensitive) when filtering animals in the spatial neighborhood.
