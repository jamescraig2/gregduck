## Phase 1: Preparation & Configuration

### 1. Verify Repo State
- Confirm no existing `package.json`, `tsconfig.json`, `next.config.*`, `node_modules/`, or prior source code.
- Confirm `.gitignore` is present (Next.js-ready).

### 2. Run `create-next-app`
```bash
npx create-next-app@latest ./ --typescript --eslint --app --no-tailwind --import-alias "@/*"
```
This generates: `package.json`, `tsconfig.json`, `next.config.ts`, `eslint.config.mjs`, `app/` (with boilerplate), `public/`, `next-env.d.ts`, `node_modules/`, and overwrites `.gitignore`.

### 3. Re-append entries to `.gitignore`
After `create-next-app` completes, append to `.gitignore`:
```
.gemini/
.agents/
```

### 4. Create custom directories
```bash
mkdir components services lib types tests
```

### 5. Install test dependencies
```bash
npm install -D vitest @vitejs/plugin-react @testing-library/react @testing-library/jest-dom jsdom
```

## Phase 3: Integration & Testing

### 1. Test Configuration ã `tests/vitest.config.ts`
```typescript
import { defineConfig } from 'vitest/config';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  test: {
    environment: 'jsdom',
    globals: true,
    setupFiles: './tests/setup.ts',
  },
  resolve: {
    alias: { '@': path.resolve(__dirname, '../') },
  },
});
```

### 2. Test Setup ã `tests/setup.ts`
```typescript
import '@testing-library/jest-dom';
```

### 3. Structural Tests ã `tests/structure.test.ts`
Test functions:
- `shouldHaveRequiredDirectories` ã verifies `app/`, `components/`, `services/`, `lib/`, `types/`, `tests/` exist
- `shouldHaveCoreFiles` ã verifies `package.json`, `tsconfig.json`, `next.config.*`, `eslint.config.*`, `.gitignore` exist
- `shouldHaveAppFiles` ã verifies `app/page.tsx`, `app/layout.tsx`, `app/globals.css` exist
- `shouldHaveTsconfigPathAlias` ã verifies `@/*` maps to `["./*"]`
- `shouldNotHaveTailwindConfig` ã verifies no `tailwind.config.*` or `postcss.config.*`
- `shouldNotHaveSrcDirectory` ã verifies no `src/` directory

### 4. Page Rendering Tests ã `tests/page.test.tsx`
Test functions:
- `rendersMainHeading` ã `screen.getByRole('heading')` contains "Gray Duck"
- `rendersTagline` ã tagline paragraph is rendered
- `rendersNoUnexpectedBoilerplate` ã no default Next.js boilerplate visible

## Phase 4: Rollout & Rollback Strategy

### 1. Rollout Steps
- No feature flags needed ã this is foundational scaffolding.
- After creation, verify with `npm run build && npm run lint && npx vitest run`.

### 2. Rollback Plan
If verification fails:
1. Remove all generated files: `rm -rf app components services lib types tests node_modules public .next`
2. Restore `.gitignore` from git: `git checkout -- .gitignore`
3. Delete generated config files: `rm -f package.json tsconfig.json next.config.ts next-env.d.ts eslint.config.mjs`
4. Revert `package-lock.json` if present: `git checkout -- package-lock.json`
