# Technical Research: Map Feature: Page Wiring & Tests (app/map/page.tsx, barrel exports, component tests) (Issue #44)

## Executive Summary

This issue focuses on connecting the map feature components (`MapDashboard`, `AnimalMarkerClusterer`, `AnimalTeaserDrawer`) into the Next.js App Router by creating `app/map/page.tsx`, expanding `components/index.ts` barrel exports, and implementing unit/integration tests for map components and page routing. All prerequisite map components and testing configurations are already in place, and the issue is ready for immediate implementation.

## Technical Challenges

- **Mocking `@vis.gl/react-google-maps` and Google Maps Globals**: `MapDashboard` and `AnimalMarkerClusterer` depend on Google Maps JavaScript API primitives and React context wrappers. Unit tests must mock `APIProvider`, `Map`, and `useMap` hooks without requiring external network calls or a live Google Maps API key during test execution.
- **Simulating Touch Swipe Events**: Testing `AnimalTeaserDrawer` downward swipe dismissal requires simulating continuous touch gestures using `fireEvent.touchStart` and `fireEvent.touchEnd` with a vertical delta ($\Delta Y > 60\text{px}$) on `data-testid="teaser-drawer"`.
- **Timer Debouncing**: `MapDashboard` debounces map camera movements by 300ms before making API calls to `/api/markers`. Tests must accurately manage fake timers (`vi.useFakeTimers()`) to verify fetch triggers after camera position updates.
