# Implementation Plan: Proximity Matching Engine (Issue #13)

## Executive Summary

This plan details the implementation of the Proximity Matching Engine layer for Issue #13. The engine uses H3 resolution 9 spatial indexing to query a 7-cell neighborhood (center cell + 6 adjacent k=1 neighbors) and determines species carrying-capacity saturation (defaulting to 3 per species). When the species count in the neighborhood is below capacity, a new animal discovery is recorded; when at or above capacity, the encounter is matched with the geographically closest animal using Haversine physical distance calculations.

     - `MatchingParams`: `{ userId: string; species: string; latitude: number; longitude: number; h3Index: string; photoUrl: string; existingAnimalsInZone: Animal[]; }`
     - `AnimalMatchingStrategy`: Interface with method `match(params: MatchingParams): Promise<MatchingResult>`

## Phase 2: Core Implementation

1. **Proximity Matching Strategy**:
   - File: `services/matching/proximity.ts`
   - Implement `ProximityMatchingStrategy implements AnimalMatchingStrategy`:
     - Validate inputs (`h3Index`, `latitude`, `longitude`).
     - Retrieve 7-cell neighborhood using `getLocalZone(h3Index)` from [`services/location/h3.ts`](file:///Users/jamescraig/gh/gregduck/services/location/h3.ts#L25).
     - Filter `existingAnimalsInZone` for matching normalized (case-insensitive) species located within the 7-cell local zone set.
     - Compare count against `getSpeciesSaturationLimit(targetSpecies)`.
     - If count < saturation limit: return `{ isNewDiscovery: true, matchedAnimal: null }`.
     - If count >= saturation limit: iterate filtered animals, calculate distance using `calculateDistanceMeters`, and return `{ isNewDiscovery: false, matchedAnimal: closestAnimal }`.
2. **Barrel Exports**:
