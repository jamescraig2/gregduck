# Technical Research: Issue 6.2: Google Maps Interface with Marker Clustering (Issue #18)

## Executive Summary

Issue #18 is **ready to implement**. All declared dependencies (Issue #6 DB schema, Issue #12 H3 spatial service, Issue #23 `GET /api/markers` endpoint) are fully implemented in the codebase. The backend markers API (`app/api/markers/route.ts`) already accepts bounding-box and H3-index queries, returns the exact marker shape the map component will consume, and has full test coverage. The implementation requires installing two new packages (`@vis.gl/react-google-maps`, `@googlemaps/markerclusterer`), building three new client components (`MapDashboard`, `AnimalMarkerClusterer`, `AnimalTeaserDrawer`), and adding one `.env` variable (`NEXT_PUBLIC_MAP_ID`).

---

## Technical Challenges

- **AdvancedMarker requires a Map ID**: `AdvancedMarker` (the modern, customizable pin) only works when a `mapId` is supplied to the `<Map>` component. The `NEXT_PUBLIC_MAP_ID` variable is absent from `.env` and `.env.example`; it must be created in the Google Cloud Console (Maps → Map IDs) and added before markers render.

- **Imperative MarkerClusterer + declarative React**: `@googlemaps/markerclusterer` is an imperative library that holds raw `google.maps.marker.AdvancedMarkerElement` references. Bridging it with React's declarative model requires using `useMap()` to get the map instance, a `useRef` for the clusterer singleton, and `useEffect` hooks to sync marker additions/removals when data changes — a pattern not currently present anywhere in the codebase.

- **No React hooks or client components exist yet**: `app/page.tsx` is a pure server component with no `useState`/`useEffect` or `"use client"` usage in any `.tsx` file. The entire map feature is a greenfield client-side feature in an otherwise mostly-server codebase.

- **Debounced viewport fetch**: `onCameraChanged` fires on every animation frame during panning/zooming. Without a debounce (≥300 ms), it will thrash the `/api/markers` endpoint. No `useCallback`/`debounce` pattern exists in the codebase to copy from.

- **IP geolocation fallback**: Browser geolocation can fail silently (permission denied, timeout). A secondary async IP-based lookup must run in parallel and resolve before a final `Minneapolis` default kicks in — requiring a Promise-race/fallback chain that is not yet established.

- **Bottom drawer touch dismissal**: Pure CSS `transform: translateY(0/100%)` handles the open/close animation, but `touchstart`/`touchmove`/`touchend` drag-to-dismiss needs raw event listeners or a lightweight scroll-snap approach; neither gesture pattern exists in the project.

- **`"use client"` boundary in App Router**: The map page must be a client component (or wrap a client component). The current `app/layout.tsx` wraps `<ClerkProvider>`, but has no existing pattern for page-level client components. Care must be taken to not accidentally make the root layout a client component.

## Technical Debt & Existing Code Patterns

### Existing Patterns

- **Glass design system** (`app/globals.css` L59–L84, `components/ui/GlassCard.tsx`, `components/ui/GlassPanel.tsx`): The project uses a well-established dark-mode glassmorphism system with CSS custom properties (`--color-bg-surface`, `--glass-backdrop-blur`, etc.). The `AnimalTeaserDrawer` should use `glass-panel` / `glass-card` classes to stay visually consistent.

- **Zod + NextRequest validation** (`app/api/markers/route.ts` L16–L66): API routes use Zod schemas to parse and transform query parameters. The `MapDashboard` component should construct URL params matching this exact schema (`sw_lat`, `sw_lng`, `ne_lat`, `ne_lng` all required together, or `h3_indexes`).

- **Vitest + MSW test setup** (`tests/api/markers.test.ts`): The project uses Vitest with `vi.mock()` for DB mocking and `@testing-library/react` for component tests. New component tests should follow this pattern.

- **H3 spatial service** (`services/location/h3.ts`): `latLngToH3Index`, `getLocalZone`, `isValidH3Index`, and `calculateDistanceMeters` are available. The map component can call `latLngToH3Index` to derive the user's current H3 cell and pass it to the viewport fetch.

- **Drizzle schema Animal type** (`lib/db/schema.ts` L44): The `Animal` type maps directly to the marker response shape returned by `/api/markers`. Define a `MarkerData` interface mirroring the API response to avoid re-deriving it.

### Technical Debt / Legacy Code

- **No `useDebounce` utility**: There is no shared debounce hook in `lib/` or `services/`. One should be created at `lib/useDebounce.ts` rather than inlining `setTimeout`/`clearTimeout` in the map component.

- **No animation library installed**: Neither Framer Motion nor any CSS animation library is in `package.json`. The bottom drawer must be implemented with vanilla CSS transitions (`transform: translateY` + `transition: 0.3s ease-out`), which is fully sufficient for this use case.

- **`components/index.ts`** only re-exports `GlassCard` and `GlassPanel`. New components (`AnimalTeaserDrawer`, map-related components) should be added to this barrel file.

- **Missing `NEXT_PUBLIC_MAP_ID` in `.env.example`**: Must be documented as a setup step.

## External Documentation & Suggested References

- **`@vis.gl/react-google-maps` official docs**: https://visgl.github.io/react-google-maps/ — `APIProvider`, `Map`, `AdvancedMarker`, `Pin`, `useMap`, `onCameraChanged` event detail shape (`ev.detail.bounds` → `{north, south, east, west}`).

- **`@googlemaps/markerclusterer` + vis.gl integration example**: https://github.com/visgl/react-google-maps/tree/main/examples/marker-clustering — Canonical imperative pattern: `useMap()` → `new MarkerClusterer({ map })` → `clusterer.addMarkers(refs)` on data change.

- **IP geolocation API (no key required)**: `https://ipapi.co/json/` returns `{ latitude, longitude, city }`. Rate-limited; cache result in component state after first call. Fallback: Minneapolis `{ lat: 44.9778, lng: -93.2650 }`.

- **Popover API for bottom sheet dismissal (modern, no-library)**: https://developer.mozilla.org/en-US/docs/Web/API/Popover_API — `popover="auto"` provides native light-dismiss. Well-supported as of 2024 (Chrome 114+, Firefox 125+, Safari 17+).

- **CSS `transform: translateY` bottom drawer**: `transform: translateY(100%)` → `translateY(0)` with `transition: transform 0.3s ease-out` — performant GPU-accelerated animation without JS libraries.

- **Google Cloud Console — Map IDs**: https://console.cloud.google.com/google/maps-apis/studio/maps — Create a "Greg Duck Dark" map ID, style it with dark theme, and copy the ID into `.env` as `NEXT_PUBLIC_MAP_ID`.

- **Next.js App Router `"use client"`**: https://nextjs.org/docs/app/building-your-application/rendering/client-components — Map components must be `"use client"`. Keep the map page wrapper a server component and import the `MapDashboard` client component into it.

## Proposed Execution Plan

### Phase 1: Preparation & Package Installation

1. Install packages:
   ```bash
   npm install @vis.gl/react-google-maps @googlemaps/markerclusterer
   ```
2. Add `NEXT_PUBLIC_MAP_ID` to `.env` and `.env.example` (Google Cloud Console → Maps → Map IDs → Create Map ID with dark theme).
3. Create `lib/useDebounce.ts` — a generic `useDebounce<T>(value: T, delay: number): T` hook using `useState` + `useEffect`.
4. Define `types/marker.ts` with `MarkerData` interface matching the `/api/markers` response shape:
   ```ts
   export interface MarkerData {
     id: string;
     name: string;
     species: string;
     lat: number;
     lng: number;
     h3Index: string;
     lastEncounteredAt: string;
     imageUrl: string;
   }
   ```

### Phase 2: Core Map Component

5. Create `components/map/MapDashboard.tsx` (`"use client"`):
   - `APIProvider` with `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`
   - `<Map>` with `mapId={NEXT_PUBLIC_MAP_ID}`, dark custom styling, `defaultZoom={13}`, `defaultCenter` from geolocation hook
   - `onCameraChanged` handler → debounced (300 ms via `useDebounce`) → extracts `ev.detail.bounds` → calls `GET /api/markers?sw_lat=...&sw_lng=...&ne_lat=...&ne_lng=...`
   - Renders `<UserPositionMarker>` and `<AnimalMarkerClusterer markers={markers} onMarkerClick={...}>`
6. Create `components/map/useGeolocation.ts` (`"use client"` hook):
   - Step 1: `navigator.geolocation.getCurrentPosition` (timeout 5s)
   - Step 2 (fallback): `fetch('https://ipapi.co/json/')` → `{ lat: data.latitude, lng: data.longitude }`
   - Step 3 (final fallback): Minneapolis `{ lat: 44.9778, lng: -93.2650 }`
   - Returns `{ center: { lat, lng }, loading: boolean }`
7. Create `components/map/AnimalMarkerClusterer.tsx` (`"use client"`):
   - `useMap()` + `useRef<MarkerClusterer>` + `useRef<Map<string, AdvancedMarkerElement>>`
   - `useEffect([map])` → instantiate `MarkerClusterer`
   - `useEffect([markers])` → diff added/removed markers, call `clusterer.addMarkers/removeMarkers`
   - Each `AdvancedMarkerElement` gets a click listener → calls `onMarkerClick(marker)`
   - Custom pin: `div` with animal `imageUrl` thumbnail + species badge, styled with glassmorphism

### Phase 3: Animal Teaser Drawer Component

8. Create `components/map/AnimalTeaserDrawer.tsx` (`"use client"`):
   - Props: `animal: MarkerData | null`, `onClose: () => void`, `userPosition: {lat, lng} | null`
   - Fixed bottom-of-screen, `max-height: 33vh`, `z-index: 1000`
   - CSS: `transform: translateY(100%)` when closed → `translateY(0)` when open, `transition: transform 0.3s ease-out`
   - Overlay `div` (full-screen, semi-transparent) → `onClick={onClose}`
   - Touch handler for drag-to-dismiss: track `touchstart` Y, on `touchend` if ΔY > 60px → `onClose()`
   - Content: `imageUrl` thumbnail, `name`, `species`, `lastEncounteredAt` (relative time), distance from user (using `calculateDistanceMeters` from `services/location/h3.ts`)
   - CTA button: `<Link href={/animals/${animal.id}}>View Full Profile</Link>`
   - Styled with `glass-panel` + project CSS custom properties

### Phase 4: Page Assembly & Routing

9. Create `app/map/page.tsx` (server component wrapper):
   ```tsx
   import { MapDashboard } from '@/components/map/MapDashboard';
   export default function MapPage() {
     return <MapDashboard />;
   }
   ```
10. Update `components/index.ts` barrel to export new map components.
