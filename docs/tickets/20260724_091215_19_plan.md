# Implementation Plan: Discovery Journal (List View) (Issue #19)

## Executive Summary

This implementation plan outlines the creation of the Discovery Journal list view at `/journal`. The page provides authenticated users with a responsive glassmorphic grid catalog of unique animal discoveries and encounter history. Key features include tabbed navigation ("My Discoveries" default vs. "All Encountered Animals"), real-time search and filter controls, URL query parameter synchronization, full-width hero photo previews with fallback image handling, relative timestamp badges, loading skeletons, infinite scrolling pagination, and unit/integration testing with Vitest.

## Phase 2: Core Page & Client Container

### 1. Client Journal Container (`components/journal/JournalContainer.tsx`)

- **Location**: `[components/journal/JournalContainer.tsx](file:///Users/jamescraig/gh/gregduck/components/journal/JournalContainer.tsx)`
- **State & Router Integration**:
  - Use `useSearchParams` and `useRouter` from `next/navigation` to read and sync `tab`, `search`, `species`, `page` state to URL query string.
  - Track `data: JournalEntry[]`, `page: number`, `totalPages: number`, `isLoading: boolean`, `isFetchingNextPage: boolean`, `error: string | null`.
- **API Fetching**:
  - Query `GET /api/journal?tab=${tab}&search=${search}&species=${species}&page=${page}&limit=20`.
  - Accumulate entries on pagination updates (page > 1) or replace list on filter/tab changes.
- **Infinite Scrolling**:
  - Use an `IntersectionObserver` attached to a bottom sentinel `div`.
  - Trigger fetch for `page + 1` when sentinel is intersecting and `page < totalPages` and not currently loading.
- **Empty & Error States**:
  - Show empty glass panel message ("No animal discoveries found matching your search criteria.") with a reset filters button.
  - Show error panel with a "Try Again" retry button on fetch failures.
