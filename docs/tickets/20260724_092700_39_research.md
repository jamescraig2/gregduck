# Technical Research: 6.1b: Camera Viewfinder, Capture Overlay & Coordinator Component (Issue #39)

## Executive Summary

This issue builds the interactive camera user interface for the wildlife discovery capture experience. It introduces three components in `components/camera/`: `CameraViewfinder`, `CaptureOverlay`, and the `CameraCaptureModal` coordinator. All prerequisites from subissue 6.1a (`lib/camera.ts` and `lib/location.ts`) are implemented and ready. The issue is fully specified and ready for implementation.

## Technical Challenges

- **Video Stream Lifecycle & Clean Unmount**: Ensuring `stopCameraStream(stream)` is reliably invoked on modal close or component unmount to release hardware resources (camera indicator LED turns off).
- **Concurrent Location & Video Stream Initialization**: `requestCameraStream()` and `getCurrentLocation()` should be initiated concurrently when `isOpen` becomes true, without blocking each other or causing race conditions during modal teardown.
- **State Machine Synchronization**: Managing state transitions (`idle`, `camera_active`, `capturing`, `locating`, `submitting`, `success`, `error`) alongside location states (`idle`, `locating`, `ready`, `error`) and ensuring shutter button disabled state cleanly maps to `isCapturing || locationStatus === 'locating'`.
- **Canvas JPEG Capture & API Submission**: Extracting the canvas frame blob via `captureFrameAsBlob(videoRef.current)` and assembling a `FormData` object with `file`, `latitude`, `longitude` fields to POST to `/api/capture`.
