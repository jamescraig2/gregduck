# Technical Research: Map Feature: Utilities & Types Setup (useDebounce, MarkerData, env vars) (Issue #41)

## Executive Summary

This issue establishes the shared foundational utilities, TypeScript type definitions, and environment configuration required for the Google Maps feature (Issue #18). The task includes creating a generic `useDebounce` hook in `lib/useDebounce.ts`, defining the `MarkerData` interface in `types/marker.ts`, re-exporting it via `types/index.ts`, and adding `NEXT_PUBLIC_MAP_ID` to `.env.example`. The task has no external issue dependencies, requires no new runtime dependencies, and is ready for immediate implementation.

## Technical Challenges

- **Timer Cleanup in `useDebounce`**: The custom hook must correctly utilize `useEffect` cleanup (`clearTimeout(timer)`) to prevent memory leaks, race conditions, or unmounted state updates during fast input changes.
- **Strict Field Alignment with API Response**: The `MarkerData` interface must match the output format of `GET /api/markers` (`id`, `name`, `species`, `lat`, `lng`, `h3Index`, `lastEncounteredAt`, `imageUrl`) to ensure type safety across frontend map components.
