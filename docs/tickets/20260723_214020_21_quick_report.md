## Executive Summary

Implemented the backend REST API endpoint `GET /api/journal` to serve authenticated user animal discovery and encounter records from the Neon Postgres database using Drizzle ORM and Clerk authentication.

---

## Key Changes

1. **`app/api/journal/route.ts`** _(New File)_
   - Implemented `GET` route handler verifying user session via Clerk `auth()`.
   - Returns HTTP `401 Unauthorized` for unauthenticated requests.
   - Computes user's first-time animal discoveries by querying user encounter history in chronological order.
   - Supports `tab` query parameter (`discoveries` for first-time unique discoveries vs `all` for full encounter timeline).
   - Supports filtering parameters: `species` (species string match), `search` (case-insensitive search on animal name or species), `startDate` / `endDate` (ISO date range filter).
   - Supports pagination: `page` (default 1) and `limit` (default 20, max 100).
   - Returns structured JSON payload matching spec with `data` array and `pagination` metadata (`page`, `limit`, `total`, `totalPages`).

2. **`tests/journal.test.ts`** _(New File)_
   - Added comprehensive unit and integration test suite covering:
     - 401 Unauthorized response for unauthenticated callers.
     - 200 OK for `tab=discoveries` returning unique first-time discoveries with `isFirstDiscovery: true`.
     - 200 OK for `tab=all` returning full encounter history with accurate `isFirstDiscovery` boolean flags.
     - Pagination handling (`page`, `limit`, `total`, `totalPages`).
     - Empty result payload handling.
     - 500 Internal Server Error when database fails.
