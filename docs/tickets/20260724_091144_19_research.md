# Technical Research: Issue 7.1: Discovery Journal (List View) (Issue #19)

## Executive Summary

This research pass evaluates the technical requirements, architecture, and implementation readiness for the Discovery Journal list view at `/journal`. The feature presents authenticated users with a responsive grid catalog of unique animal discoveries and encounter history, incorporating tabbed navigation ("My Discoveries" default vs. "All Encountered Animals"), real-time search/filter controls, large photo hero headers, relative timestamp badges, loading skeletons, and infinite scrolling pagination. All prerequisite backend dependencies—including database schemas (Issue #6), Clerk authentication (Issue #7), and the `GET /api/journal` endpoint (Issue #21)—are fully implemented and verified with unit/integration test suites, confirming this issue is ready for immediate implementation.

## Technical Challenges

- **URL Query Parameter State Synchronization**: The UI must maintain tab choices (`tab=discoveries` | `tab=all`), search terms (`search`), species filters (`species`), date range bounds (`startDate`, `endDate`), and sort modes (`sort`) seamlessly synchronized with browser URL query parameters. Changing tabs or search inputs must push/replace URL states without causing full page re-renders or layout shifts.
- **Client-Side vs. Server-Side Data Fetching**: Next.js App Router pages at `/journal` can leverage React Server Components or Client Components. Since tab toggles, search inputs, and infinite scroll require dynamic interactivity, the root journal view needs a Client Component architecture or `useSearchParams` / `useRouter` integration with proper `<Suspense>` boundaries to avoid SSR hydration mismatches.
- **Responsive Multi-Column Card Layout with Image Fallbacks**: Cards require full-width hero photo previews with lazy loading and placeholder fallbacks (e.g. SVG pattern or fallback asset) when user photo URLs or species photo URLs are null or fail to load. Grid layouts must adapt dynamically across breakpoint triggers (1 column mobile, 2-3 columns tablet, 4 columns desktop).
- **Pagination & Infinite Scroll Thresholds**: Supporting both standard page navigation and infinite scroll requires tracking page index metadata (`page`, `totalPages`), managing appending logic for infinite scroll without duplicate item keys, and preserving scroll position when switching tabs.

   - Create `components/journal/JournalHeader.tsx`: Tabbed navigation bar defaulting to "My Discoveries" (`tab=discoveries`), tab toggle buttons ("My Discoveries", "All Encountered Animals"), search bar input, date range pickers, and sort selector.
   - Create `components/journal/AnimalCard.tsx`: Responsive card featuring full-width hero image preview with image error fallback, animal name heading, species badge, relative timestamp format, and "First Discovery" status badge.

2. **Phase 2: Page Route & Infinite Scroll Integration**
   - Create `app/journal/page.tsx`: Server page route wrapped in `<Suspense fallback={<JournalGridSkeleton />}>` delegating to `components/journal/JournalContainer.tsx`.

   - Implement infinite scrolling using Intersection Observer on the bottom sentinel element to fetch subsequent pages.
3. **Phase 3: Testing & Verification**
   - Create test suite `tests/journal_page.test.tsx` using `@testing-library/react` and Vitest.
   - Test tab switching updates API fetch query params, search input filtering, loading state skeletons, empty states ("No discoveries logged yet"), and error retry buttons.
