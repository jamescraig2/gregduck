# Technical Research: 6.1c: Discovery & Encounter Result Modals + Component Test Suite (Issue #40)

## Executive Summary

This technical research document analyzes the requirements for issue #40, which encompasses building two glassmorphic modal components (`DiscoveryModal` and `EncounterModal`) and establishing a comprehensive Vitest + React Testing Library + MSW component test suite for the camera capture workflow. All requirements, component signatures, accessibility guidelines (WAI-ARIA dialog specifications), and test assertions are well-defined and ready for implementation.

## Technical Challenges

- **Modal Overlay & Glassmorphic Styling**: Both modals require fixed full-screen overlays with slide-up and fade-in transitions. `<GlassPanel>` from `components/ui/GlassPanel.tsx` serves as the container, relying on CSS design tokens defined in `app/globals.css`.
- **WAI-ARIA Dialog Accessibility Compliance**: Modals must strictly follow accessibility standards with `role="dialog"`, `aria-modal="true"`, and `aria-labelledby` pointing dynamically or deterministically to the modal title ID (`headline-discovery` / `headline-encounter`).
- **Date Formatting**: `EncounterModal` receives an ISO timestamp string (`capturedAt`) and must format it into a clean, human-readable date string (e.g. using `new Date(capturedAt).toLocaleDateString()`).
- **Vitest + RTL Browser APIs Mocking**: `tests/components/CameraCapture.test.tsx` tests the camera modal lifecycle and requires stubs for `navigator.mediaDevices.getUserMedia`, `navigator.geolocation.getCurrentPosition`, and `HTMLCanvasElement.prototype.toBlob`.

     2. `DiscoveryModal renders headline and animal name when isOpen is true`

     11. `CameraCaptureModal calls onCaptureComplete with parsed result on 2xx`
     12. `CameraCaptureModal calls onClose after successful capture`
   - Run `npm test -- tests/components/CameraCapture.test.tsx` and ensure zero TypeScript errors (`npx tsc --noEmit`).
