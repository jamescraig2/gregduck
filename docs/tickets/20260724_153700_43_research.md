# Technical Research: Map Feature: Animal Marker Clusterer & Teaser Drawer (Issue #43)

## Executive Summary

Issue #43 is **ready to implement**. All prerequisites from subissues #41 (Utilities & Types) and #42 (Core Map Component & Geolocation) are present in the codebase. Specifically, `MarkerData` is exported from `types/index.ts`, `calculateDistanceMeters` is exported from `services/location/h3.ts`, `@googlemaps/markerclusterer` and `@vis.gl/react-google-maps` are installed in `package.json`, and `MapDashboard.tsx` is ready to pass `markers`, `selectedAnimal`, `setSelectedAnimal`, and user location `center` into the target components.

Implementation requires replacing the stub components in `components/map/AnimalMarkerClusterer.tsx` and `components/map/AnimalTeaserDrawer.tsx` with full client-side implementations.

## Technical Challenges

- **Imperative MarkerClusterer React Lifecycle**: `@googlemaps/markerclusterer` uses an imperative JavaScript API rather than React virtual DOM nodes. `AnimalMarkerClusterer` must use `useMap()` to acquire the Google Maps `map` instance, construct a `MarkerClusterer` instance in `useEffect`, and maintain a `Map<string, google.maps.marker.AdvancedMarkerElement>` ref to diff incoming `markers` props so markers are added/removed without unnecessary re-creation or memory leaks.
- **AdvancedMarker Custom Pin Rendering**: Custom DOM elements for `AdvancedMarkerElement` must be instantiated (`document.createElement('div')`) with glassmorphism styling, a thumbnail `<img>`, species badge, and proper event listeners attached for marker clicks (`onSelect`/`onMarkerClick`).
- **Touch Drag-to-Dismiss Gesture**: `AnimalTeaserDrawer` requires a custom touch gesture handler (`touchstart` & `touchend` event listeners or React synthetic touch handlers) tracking vertical delta $\Delta Y$. If $\Delta Y > 60\text{px}$, it triggers `onClose()`.
- **Distance Calculation & Nullable User Position**: `userPosition` passed from `useGeolocation` via `MapDashboard` may be null while loading. The drawer must gracefully display distance when `userPosition` is present via `calculateDistanceMeters(userPosition.lat, userPosition.lng, animal.lat, animal.lng)` and hide/omit distance display when `userPosition` is null.

## Technical Debt & Existing Code Patterns

### Existing Patterns

- **`calculateDistanceMeters`**: Located in `services/location/h3.ts`. Accepts `(lat1, lng1, lat2, lng2)` and returns distance in meters.
- **Glassmorphism UI Primitives**: CSS utility class `glass-panel` is available in `app/globals.css`.
- **`MarkerData` Type**: Exported from `types/index.ts` with fields: `id`, `name`, `species`, `lat`, `lng`, `h3Index`, `lastEncounteredAt`, `imageUrl`.
- **`MapDashboard` Component**: Located in `components/map/MapDashboard.tsx`. Renders `<AnimalMarkerClusterer>` and `<AnimalTeaserDrawer>`. Needs minor update to pass `onClose` and `userPosition`.

### Technical Debt / Cleanup Needed

- **`AnimalMarkerClusterer.tsx` & `AnimalTeaserDrawer.tsx`**: Currently return `null` (stubs). Must be updated with real components and updated prop interfaces (`AnimalTeaserDrawerProps` needs `onClose: () => void` and `userPosition?: { lat: number; lng: number } | null`).
- **MapDashboard integration wiring**: `MapDashboard.tsx` needs to pass `onClose={() => setSelectedAnimal(null)}` and `userPosition={center}` to `<AnimalTeaserDrawer />`.

## Proposed Execution Plan

1. **Phase 1: Implement `components/map/AnimalMarkerClusterer.tsx`**
   - Props interface: `{ markers: MarkerData[]; onSelect: (animal: MarkerData | null) => void; }`.
   - `useMap()` hook to access the map instance.
   - Ref for `MarkerClusterer` instance and `Map<string, google.maps.marker.AdvancedMarkerElement>` for marker mapping.
   - Diff marker list to create custom DOM pin elements with glassmorphism style, image, species label, and click handler calling `onSelect(marker)`.
2. **Phase 2: Implement `components/map/AnimalTeaserDrawer.tsx`**
   - Props interface: `{ animal: MarkerData | null; onClose: () => void; userPosition?: { lat: number; lng: number } | null; }`.
   - Fixed bottom-sheet drawer (`max-height: 33vh`, `z-index: 1000`) with smooth CSS transform translation (`translateY(0)` vs `translateY(100%)`).
   - Semi-transparent backdrop overlay with `onClick={onClose}`.
   - Touch event handlers (`onTouchStart`, `onTouchEnd`) tracking $\Delta Y > 60\text{px}$ to invoke `onClose()`.
   - Content: image thumbnail, name, species, relative timestamp, distance calculated via `calculateDistanceMeters`, and Next.js `<Link href={\`/animals/\${animal.id}\`}>`.
3. **Phase 3: Update `MapDashboard.tsx` Wiring**
   - Update `<AnimalTeaserDrawer animal={selectedAnimal} onClose={() => setSelectedAnimal(null)} userPosition={center} />`.
4. **Phase 4: Unit Testing**
   - Add component unit tests covering drawer visibility, drag gesture close, distance formatting, and marker selection.
