# Technical Research: Map Feature: Core Map Component & Geolocation Hook (MapDashboard, useGeolocation) (Issue #42)

## Executive Summary

Issue #42 is **ready to implement**. All declared dependencies from the "Utilities & Types Setup" sub-issue (#41) are fully satisfied in the current codebase: `useDebounce<T>` lives in `lib/useDebounce.ts`, `MarkerData` is defined in `types/marker.ts` and re-exported via `types/index.ts`, and `NEXT_PUBLIC_MAP_ID` is documented in `.env.example` for developer setup. The implementation requires creating two new files (`components/map/useGeolocation.ts` and `components/map/MapDashboard.tsx`) and installing two npm packages (`@vis.gl/react-google-maps`, `@googlemaps/markerclusterer`). The backend `GET /api/markers` endpoint is fully operational and returns the exact `MarkerData` shape the component will consume.

## Technical Challenges

- **Three-Step Geolocation Fallback Chain**: The `useGeolocation` hook must handle three distinct async paths: (1) `navigator.geolocation.getCurrentPosition` with 5000 ms timeout, (2) HTTP fetch to `https://ipapi.co/json/` extracting `data.latitude`/`data.longitude`, and (3) hardcoded Minneapolis default `{ lat: 44.9778, lng: -93.2650 }`. Each step must only fire if the prior step fails, with race-condition-safe cleanup (`useEffect` cleanup on unmount).
- **Debounced Bounds Fetch**: `onCameraChanged` fires on every frame during pan/zoom. The implementation must store `ev.detail.bounds` in state, pass it through `useDebounce(bounds, 300)`, and only issue `GET /api/markers?...` when the debounced value changes — guarded with a `useEffect` dependency on `debouncedBounds`.
- **AdvancedMarker requires Map ID**: `AdvancedMarker` (needed for clustering) only works when a `mapId` is supplied to `<Map>`. `NEXT_PUBLIC_MAP_ID` must be set in `.env.local` before markers render; it is absent from the live `.env` file.
- **Stub Components**: `<AnimalMarkerClusterer>` and `<AnimalTeaserDrawer>` do not yet exist and must be stubbed as pass-through components to allow `MapDashboard` to compile.

2. **Phase 2: Create `components/map/useGeolocation.ts`** (`"use client"`)
